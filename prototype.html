REST API reference and integration guides

Getting Started

The GTM Agent Coordinator provides a REST API and MCP server for coordinating AI agent actions. All API requests require authentication via API key.

📖
Full Documentation: Visit api.leandata.com/docs for complete API reference with interactive examples.

Base URL

https://api.leandata.com/v1

Authentication

Include your API key in the Authorization header:

Authorization: Bearer gtmc_live_sk_...
Preflight API
Agents API
Records API
Scheduling API
MCP Server

POST /v1/preflight

Core Capability

Check whether a proposed action on a GTM record is allowed. This is the primary coordination endpoint that all agents must call before taking any action.

Request Example

POST /v1/preflight Authorization: Bearer gtmc_live_sk_... { "record_identifiers": { "email": "jane@bigcorp.com", "sfdc_lead_id": "00Q1234567890" }, "proposed_action_type": "send_email", "proposed_action_detail": { "subject": "Quick question about routing", "sequence_name": "Enterprise Outbound v3" }, "urgency": "normal" }

Response Example (GO)

{ "request_id": "req_01HXYZ...", "decision": "GO", "reason_code": "NO_CONFLICTS_DETECTED", "reason_text": "No conflicts or policy restrictions found. Proceed with action.", "current_owner": { "name": "Mike Chen", "sfdc_id": "005ABCDEF", "role": "Account Executive" }, "record_context": { "canonical_record_id": "rec_01H...", "account_name": "BigCorp", "has_open_opportunity": true }, "latency_ms": 127 }

Response Example (NO_GO)

{ "request_id": "req_01HXYA...", "decision": "NO_GO", "reason_code": "BUDGET_EXCEEDED", "reason_text": "Communication budget exceeded: 3/3 emails used in 7-day window", "budget_status": { "emails_used": 3, "emails_limit": 3, "window": "7d_rolling", "resets_at": "2026-06-20T08:15:00Z" }, "latency_ms": 98 }

Decision Types

  • GO - Proceed with the action
  • NO_GO - Do not proceed (policy block, budget exceeded, etc.)
  • WAIT - Retry later (cooldown period, collision detected)
  • REDIRECT - Take a different action or route to a different owner

Agent Registration API

POST /v1/agents

Register a new agent and receive an API key.

POST /v1/agents Authorization: Bearer { "agent_name": "11x Alice - Outbound SDR", "agent_type": "ai_sdr", "declared_action_types": ["send_email", "send_linkedin", "schedule_meeting"], "source_system": "11x" } Response: { "agent_id": "ag_01HXYZ...", "api_key": "gtmc_live_sk_...", // Only returned once! "status": "active" }

GET /v1/agents

List all registered agents.

PATCH /v1/agents/:agentId/status

Change agent status (active/paused/revoked).

Record Identity Resolution API

POST /v1/records/resolve

Resolve a person's identity from partial identifiers.

POST /v1/records/resolve { "email": "jane@bigcorp.com", "name": "Jane Doe", "company": "BigCorp" } Response: { "canonical_record_id": "rec_01H...", "match_type": "exact_email", "match_confidence": 100, "record": { "primary_email": "jane@bigcorp.com", "full_name": "Jane Doe", "company_name": "BigCorp", "current_owner": { ... }, "account": { ... } } }

Agentic Scheduling API

POST /v1/scheduling/availability

Fetch available meeting slots for a record's designated host.

POST /v1/scheduling/availability { "canonical_record_id": "rec_01H...", "meeting_type": "discovery_call", "duration_minutes": 30, "date_range": { "start": "2026-06-16", "end": "2026-06-20" } } Response: { "host": { "name": "Mike Chen", "email": "mike.chen@acme.com" }, "available_slots": [ {"start": "2026-06-16T10:00:00-07:00", "end": "2026-06-16T10:30:00-07:00"}, ... ] }

POST /v1/scheduling/book

Book a meeting at a specific time slot.

Model Context Protocol (MCP) Server

The Coordinator provides an MCP server for LLM-native agent integration. MCP tools provide a natural language interface to all coordination capabilities.

MCP Server URL

https://mcp.leandata.com/v1

Available Tools

  • preflight_check - Check if an action is allowed
  • resolve_record - Find a person's canonical record
  • get_ownership - Get record owner and context
  • fetch_availability - Get meeting availability slots
  • book_meeting - Book a confirmed meeting
  • query_ledger - Query recent agent activity

Example: Preflight Check via MCP

// Agent calls MCP tool await mcp.call_tool("preflight_check", { record_email: "jane@bigcorp.com", proposed_action_type: "send_email", proposed_action_detail: "Follow-up on demo" }) // Returns natural language response + structured data "✅ GO - You can proceed with sending an email to jane@bigcorp.com. No conflicts detected. Record is owned by Mike Chen (AE) and has an active $180K opportunity in Negotiation stage."
